# Guides / Construct / Anchor vocabulary — and Guides feed PIC, never bypass it

Construction scaffolding lands in the Builder (spec: `CONSTRUCTION_GUIDES_SPEC.md`). Canonical nouns: **Construct** is the Design-phase mode (beside Place and Complete); **Guide** is a drawn construction element (Guide line, Guide circle, divided Guide circle) — "construction line" is the informal/literature synonym only; **Anchor** is promoted to the *umbrella term* for every single point pickable for placement or Complete, app-wide (exposed vertices, Frame nodes, boundary-section points, neighbour vertices, Guide anchors), consolidating four flows that each named their points differently. Guide/Anchor deliberately avoid colliding with **Ray** / **Strand** / **Contact Ray**.

The boundary decision worth recording: **Guides produce Tiles, never pattern lines.** The traditional compass-and-straightedge method draws the pattern itself; we rejected that (a second rendering pipeline bypassing PIC) — Guides expose Anchors, Anchors feed the existing Place/Complete flows, and PIC remains the single engine. Free-standing anchor-only Completes are allowed (scaffold-first workflow), riding the flexible-placement overlap gate. If direct pattern-line authoring is ever wanted, it belongs to the nested-layers idea, not Guides.
